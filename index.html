1. Who operates Tetora

Tetora is an independently operated Discord bot. Privacy questions and data requests can be sent to zaelrae@pm.me. Include your Discord user ID and the relevant server ID so the correct records can be located without collecting unnecessary identity documents. Never send a Discord password, authentication token, or identity document with a request.

2. Data Tetora processes

  • Discord identifiers and member details: server, user, channel, role, and message IDs; usernames or display names; role membership; join, leave, boost, and account timing information needed for configured server features.
  • Server configuration: feature settings, selected channels and roles, panel configuration, FAQs, scheduled announcements, and similar administrator-provided content.
  • Feature records: XP, levels, achievements, economy balances and transactions, giveaways and entries, ticket status and channel references, moderation cases, verification status, and audit details such as the responsible moderator and timestamp.
  • Message content: content, links, embeds, and attachment indicators may be examined when needed for commands, XP, moderation and cross-channel anti-spam rules, bump reminders, summaries, or another enabled feature.
  • Moderation evidence: when automation escalates an uncertain incident for human review, relevant message excerpts and their channel/message references may be stored with the case.

Ordinary message content is not added to Tetora's off-platform database. Messages and embeds that Tetora posts inside Discord, including ticket and staff-facing messages, remain subject to that server's permissions and Discord-side deletion.

3. Privileged Discord intents

Server Members

Tetora uses member join, update, and leave events for configurable welcome and goodbye messages, initial and level roles, staff-role pairing, booster benefits, member lifecycle handling, and join-raid detection. These features need to react to membership and role changes even when the affected member does not run a command.

Message Content

Tetora uses message content for configured moderation and anti-abuse protection, including cross-channel correlation, and for features that inherently depend on a message, such as XP, bump detection, and deliberately invoked AI summaries. Most content is handled only long enough to evaluate the event or complete the request. Excerpts are persisted only when an incident is escalated for human moderation review and are automatically redacted within 30 days.

Presence

Tetora does not request the Presence intent and does not track online status, activities, or games played.

Moderation is a server-level safety function and cannot be individually disabled while a person participates in a server where it is enabled. Optional features such as AI commands run only when deliberately invoked, and server administrators can disable or reconfigure Tetora features.

4. Age verification

If a server uses Tetora's 18+ verification workflow, an authorised moderator may provide a date of birth to determine whether the member is 18 or older. Tetora uses that value only to make the age decision and does not store the date of birth. It retains the verification result, relevant Discord IDs, display name, verifier, and verification time for server administration and audit purposes. Images or identity documents used by moderators are not uploaded to or stored in Tetora's database.

5. How the data is used

Data is used only to provide and secure the features configured by each Discord server: moderation and anti-abuse protections, roles and member lifecycle actions, support tickets, giveaways, announcements, FAQs, economy and XP systems, settings, and requested bot responses. Database records and configuration are scoped by Discord server and are not combined into advertising or cross-server user profiles.

6. AI-assisted features

When someone deliberately uses an AI command, the prompt and any context needed for that request may be sent to the operator-configured, self-hosted language-model service. For example, a summary request needs the selected recent messages. Tetora does not use message content to train AI or machine-learning models, and the bot does not persist ordinary AI prompts or responses in its database. The model service is hosted on operator-controlled infrastructure rather than being used for advertising or data brokerage.

If a recovery backup still contains data already removed from the live database, that residual copy remains unavailable to normal bot operations and is removed when the backup is overwritten or expires. A deletion must be reapplied if such a backup is ever restored.

8. Sharing and disclosure

Tetora does not sell personal data, build advertising profiles, or share data with data brokers. Data is handled through Discord and operator-controlled infrastructure, including the self-hosted AI service when an AI feature is invoked. It may also be disclosed when reasonably necessary to comply with law, protect users, investigate abuse, or secure the service.

This public policy is hosted using GitHub Pages. The page contains no Tetora-operated analytics, advertising, cookies, or third-party scripts, although GitHub may process access information under its own privacy statement.

9. Security

Access to Tetora's host and database is restricted to the operator and the bot's dedicated operating-system account. The active database is encrypted at rest and protected by restricted filesystem access. Administrative access is limited, and safeguards are reviewed as Tetora's deployment and features change.

Encryption at rest protects the storage while it is locked; it is not end-to-end encryption and does not replace host, account, Discord-permission, or network security. The operator also limits retained data and applies automated redaction to moderation evidence. No internet service can guarantee absolute security.

10. Your choices and data requests

To ask what data Tetora has about you, correct it, or request deletion, email zaelrae@pm.me with the subject Tetora data request, your Discord user ID, the relevant server ID, and the type of request. Server administrators may also request deletion of their server's configuration and feature records.

The operator will respond within a reasonable period and may ask for a safe method of verifying control of the relevant Discord account or server. A request may be limited where records are still reasonably needed for security, moderation, fraud prevention, dispute handling, or a legal obligation. Any such limitation will be explained where permitted.

11. Children

Tetora is not directed to children under the minimum age required to use Discord. Individual servers may set higher age requirements. If you believe a child has provided data to Tetora contrary to those requirements, contact the operator.

12. Discord and policy changes

Discord independently processes information under its own Privacy Policy; this document covers Tetora's handling of data and does not replace Discord's policy. This policy may change as Tetora's features, safeguards, or legal obligations change. Material updates will be posted on this page with a revised date.
